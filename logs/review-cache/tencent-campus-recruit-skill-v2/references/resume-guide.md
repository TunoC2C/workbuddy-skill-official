# 腾讯校招简历优化指南

> 本材料基于腾讯招聘团队分享整理，聚焦方法论和面试官视角，不硬编码具体岗位信息。

## 一、写简历前的思考

### 对话引导

```
写简历之前，建议你先想清楚三个问题：

1. 你要投的岗位 JD 说了什么？
   → 告诉我岗位名/方向/城市，我优先调 JD 知识库（campus-recruit-jd-qa）拉最新 JD；如果 MCP 不可用，就用 `scripts/fetch_recruit_jds.py` 从官网拉取真实 JD，再帮你拆解

2. 你的经历和这个岗位匹不匹配？
   → 不只是实习经历，你的项目、竞赛、个人特色都算

3. 你有什么独特的亮点？
   → 这个很重要！面试官在成百上千份简历里找"不一样"的那个人
```

### 深挖个人亮点（最关键的建议）

来自腾讯面试官的分享：
> "筛简历时不仅看实习和学历，还会搜索一些个人特色的关键词。
> 比如有的面试官会搜'游戏经历'，看谁写了自己的游戏成就和玩法思考。
> 这些个人特色很有机会被识别到。"

腾讯鼓励多元化，每个团队想招"适配"的人。**展示真实的你和你的热爱，比千篇一律的包装更有效。**

## 二、STAR 法则

用 STAR 结构描述项目经历，让描述有逻辑、有层次：

| 要素 | 含义 | 关键 |
|------|------|------|
| **S** - Situation | 情境/背景 | 面对什么问题？ |
| **T** - Task | 任务/职责 | 你承担了什么？ |
| **A** - Action | 行动/方法 | 你具体做了什么？ |
| **R** - Result | 结果/成果 | 带来了什么量化效果？ |

### 对话引导

```
来试一下，拿你比较有代表性的一段经历：

先别急着写，回答我四个问题——
1. 当时面对什么问题或挑战？（这就是S）
2. 你在里面负责什么？（这就是T）
3. 你具体做了什么？关键动作是什么？（这就是A）
4. 最后效果怎么样？有没有数据可以量化？（这就是R）

回答完这四个问题，一段好的项目描述就出来了。
```

## 三、常见简历误区

| 误区 | 问题 | 改进方向 |
|------|------|---------|
| **信息罗列没有重点** | 面试官扫一眼不知道你的核心竞争力 | 突出与目标岗位最相关的信息 |
| **大量堆砌缺少逻辑** | 什么都写了但逻辑不清 | 结构化：做了什么→亮点→成果 |
| **描述空洞** | 只说"负责了XX" | 用具体事例和数据支撑 |
| **缺少数据支撑** | 只说做了什么没说效果 | 量化成果："提升XX%""覆盖XX用户" |
| **一份简历走天下** | 不同岗位用同一份 | 根据目标岗位调整侧重点 |
| **逻辑不清晰** | 读完不知道你在说什么 | 理清因果关系，按重要性排列 |

## 四、常见表达优化

| 原始表达 | 优化方向 | 原因 |
|---------|---------|------|
| "参与了XX项目" | 明确你的具体职责和成果 | 缺少具体贡献和量化结果 |
| "熟悉XX技术" | 补充使用场景和实际案例 | 缺少场景佐证 |
| "沟通能力强" | 用具体协作事例证明 | 空洞声称不如事实佐证 |
| "负责XX开发" | 补充解决的问题和效果指标 | 缺少背景和成果 |

## 五、分方向简历侧重建议

> 注意：具体岗位名称和要求请以 join.qq.com 最新JD为准。以下仅为通用的方向性建议。

### 技术方向
- 项目经历是核心（用STAR描述）
- 技术栈要列清楚，但不要只列名字
- 竞赛/论文/开源贡献是加分项
- 如有条件，附代码仓库链接

### 产品方向
- 突出产品思维和用户洞察
- 数据分析案例比泛泛的职责描述更有说服力
- 展现你对产品方向的真实热情

### 游戏方向
- **游戏经历和深度体验往往是面试官很看重的**
- 做过的游戏demo、用过的引擎要突出写
- 展示你对游戏品类的深入理解

### 设计方向
- 作品集链接是必备项
- 展示设计思维过程，不是只展示成品

### 市场/职能方向
- 细节执行力 + 实际产出
- 展示学习心态和探索精神

## 六、格式建议

- 一页PDF为宜
- 简洁字体，清晰排版
- 倒序时间排列经历
- 避免花哨装饰
- 联系方式确保正确

## 九、正直原则（红线）

简历辅导必须坚守正直底线，这是不可逾越的红线：

- **禁止编造**：不能帮同学虚构项目经历、实习经验、技能证书等任何内容
- **禁止夸大**：不能刻意引导同学夸大项目成果、个人贡献、数据指标
- **如实优化**：只能在同学真实经历的基础上，帮助更好地表达和结构化
- **区分优化与造假**："这段经历可以更好地提炼亮点" ≠ "可以把数据写大一点"

面试官有丰富的识别经验，夸大或编造的内容在面试深挖时会暴露，反而得不偿失。

---

## 十、AI 技能加分引导

如果同学具备 AI 应用相关技能，主动引导其在简历中更好地呈现：

- **具体化**：不要只写"熟悉 AI 工具"，要写具体用了什么工具、解决了什么问题、效果如何
- **场景化**：将 AI 技能与岗位需求结合表述（如 prompt engineering、大模型应用开发、AI 辅助数据分析等）
- **成果化**：量化 AI 工具带来的效率提升或业务价值

如目标岗位或官方 JD 中出现 AI 相关要求，具备真实 AI 应用能力的同学可以在简历中重点呈现这一亮点。
