# 求职记忆工作流

## 目标

在用户持续对话时，维护一份本地工作空间求职画像，用于后续岗位推荐、简历优化、面试辅导和跟进提醒。记忆只服务于求职建议，不用于判断录用概率或输出敏感承诺。

## 默认记忆文件

- 默认路径：`career-memory/campus-recruit-memory.md`
- 可通过环境变量 `CAREER_MEMORY_FILE` 覆盖
- 文件不应随 skill 上架包分发，不应提交到公开仓库

## 允许记录

仅记录用户明确提供、且会影响求职建议的长期信息：

- 学历阶段、年级、毕业时间、专业方向
- 目标岗位方向、目标城市、招聘类型
- 当前招聘阶段：待投递、已投递、笔试、面试、offer/签约、实习中等
- 可实习时间、到岗时间、每周可实习天数
- 技能栈、项目/实习亮点、作品集类型
- 简历/面试辅导中已确定的改进方向
- 用户偏好的沟通风格或输出格式

## 禁止记录

不要写入以下内容：

- 身份证号、手机号、住址、银行卡、验证码、账号密码、API Key、Cookie、Token
- 完整聊天原文、附件全文、未脱敏简历全文
- 薪酬数字、offer 档位、内部评价、HC/通过率/录取率等敏感信息
- 未经用户明确提供的推断信息，例如「性格」「家庭背景」「经济状况」

## 使用规则

1. 对话开始或进入个性化建议前，如存在记忆文件，先读取并只使用与当前问题相关的信息。
2. 当用户提供稳定信息时，可以在回答后更新记忆；只写结构化摘要，不写原文。
3. 更新记忆前不需要打断对话反复确认，但如内容接近隐私边界，应先询问是否保存。
4. 用户说「忘记我」「删除记忆」「不要保存」时，立即停止新增，并按用户要求删除或编辑记忆文件。
5. 记忆与官方数据冲突时，以官方知识库、官网岗位和用户当前最新表述为准。

## 推荐更新格式

```markdown
- 2026-05-03：目标岗位方向更新为「后台开发/平台工程」；优先城市「深圳/北京」；当前阶段「准备暑期实习投递」。
```

## 脚本辅助

```bash
python scripts/career_memory.py init
python scripts/career_memory.py show
python scripts/career_memory.py append --text "目标岗位方向：后台开发；优先城市：深圳/北京"
python scripts/career_memory.py forget
```
