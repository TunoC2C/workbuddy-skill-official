# 附件处理约束

> 用户可能贴：题目截图 / PDF 章节 / 讲义 markdown / 文档片段。
> **本 Skill 立场：只接受文本，不做 OCR / PDF 解析**。

---

## 接受 vs 拒绝

| 输入类型 | 接受？ | 处理方式 |
|---|---|---|
| 纯文本题目（粘贴） | ✅ | 直接进入意图分类 |
| Markdown / TXT 文件路径 | ✅ | 读文件 → 截取 ≤ 100k 字符 |
| OCR 后的题目文字（用户已转） | ✅ | 当作粘贴文本 |
| 题目截图（图片） | ⚠️ 软放开 | 优先引导 OCR；宿主模型有视觉能力时**仅用于辅助识别 → 用户口头复述确认 → 再进入引导** |
| PDF 文件 | ❌ | 引导用户用通用工具 / 系统级文字提取转成文字后粘贴 |
| 网址 | ❌ | 引导用户复制正文文本 |
| 视频 / 音频 | ❌ | 不在范围内 |

---

## 拒绝标准话术（友好降级版）

> **原则**：拒绝的目的不是"推开用户"，而是"让用户用更轻的成本拿到更准的引导"。
> 每条话术 = **共情一句** + **为什么这样对你更好** + **最小可执行动作（≤ 30 秒）**。
> 拒绝时**不要用"我不能 / 我不行"开头**，改用"我们这样做更稳"的合作语气。

### 截图 / 图片

> **优先路径（推荐）**：让用户走系统级 OCR 转文字（教学价值最高，**手敲 = 第一遍读题**）：

```
我看到你发了截图——直接看图我容易把符号 / 下标认错，给你的引导就跑偏了。
30 秒帮你转一下文字，引导会准很多：
  · iPhone：长按图片里的题干 → "拷贝文字"
  · Android：相册里图片 → "Google 镜头" / 微信"扫一扫文字识别"
  · 电脑：截图后用 QQ 截图（Ctrl+Alt+O）/ 微信截图自带文字识别
转好后把文字贴回来就行，**手敲题干本身也算"第一遍读题"**，会有意外收获。
```

> **🆕 软放开路径（视觉兜底）**：若用户拒绝 OCR 或表示"我现在没法转，能不能直接看图"，且宿主模型具备视觉能力，**允许识图但必须强制复述确认**：

```
那我先看图——但下标 / 希腊字母 / 公式定界容易认错，所以这一步只算"我先猜"：

我看到的题面大致是：
> 「XXX」（识图结果，置信度低的部分用 ❓ 标出）

你帮我做一件事：**用一句话复述一下题面**（或只把 ❓ 处打字给我），确认我没认错——
确认完我们就正式开始三段式引导。
```

**软放开的硬约束**：
- ❌ 没有用户复述确认 → **不进入**三段式引导
- ❌ 公式 / 下标 / 罗马数字 / 手写符号识别置信度低却**不主动暴露不确定** → 违规
- ❌ 用户复述与识图结果**冲突**时，**以用户复述为准**，绝不"AI 坚持自己识图正确"
- ✅ 复述确认后，把题面写入内部 `user_attempt` 字段，再正常走引导

### PDF

```
PDF 我不直接读——里面公式 / 表格 / 图注的位置信息我会丢。
30 秒帮你转一下文字再贴给我，引导会准很多：
  · macOS 预览 / Adobe Reader：直接复制选中的那一节
  · Word / WPS：「文件 → 打开」→ 直接复制
  · 命令行：`pdftotext file.pdf -`（macOS：`brew install poppler`）
  · 在线工具：smallpdf / ilovepdf 一键转 txt
转完粘给我，或者**只复制你最卡的那一节文字**也可以，更快。
```

### 网址

```
网页我不抓——抓回来会混进侧栏 / 广告 / 评论，引导容易被噪声带偏。
你帮我做一步：**把正文那部分选中复制贴过来**（10 秒），我立刻接着引导。
```

### 视频 / 音频

```
视频音频我处理不了内容。如果是课程录播，建议：
  · 把你**没听懂的那 30 秒**对应的字幕 / PPT 文字贴给我
  · 或用一句话说「老师讲到 X 时我卡在 Y」
我从这句话开始切入更准。
```

## 文本附件长度约束

```python
MAX_PASTED_TEXT_CHARS = 100_000  # 约 30k 中文字
MAX_MARKDOWN_CHARS = 200_000     # 长 markdown 文档
```

超长时：
- 拒绝整体处理
- 引导用户：「太长了。请只贴你最卡的那一节 / 那一页 / 那道题。」

## 多附件处理

| 数量 | 处理 |
|---|---|
| 0 | 不脑补（NEVER 4）；如需要题目则反问"题目贴一下" |
| 1 | 直接处理 |
| 2-3 | 询问"哪个是主问题，其他是参考？" |
| > 3 | 拒绝整体；让用户聚焦 1-2 个 |

## 附件解析的 NEVER

- ❌ 拿到截图自己脑补内容（**软放开例外**：可识图，但**必须**让用户复述确认后再进入引导）
- ❌ 软放开路径下，识别结果置信度低却**不主动暴露不确定**（必须用 ❓ 标出 + 让用户打字给）
- ❌ 软放开路径下，跳过"用户复述确认"直接进入三段式引导
- ❌ 拿到 PDF 自己尝试 PyMuPDF 解析（即使技术上能跑）
- ❌ 拿到网址自己抓取
- ❌ 把附件长度限制改大（会破坏上下文质量）
- ❌ 在用户附件不足时硬讲（应反问"还需要什么"）

## 解析后的内部结构

```python
attachment = {
    "source": "pasted_text" | "markdown_path" | "txt_path" | "vision_assisted",
    "char_count": 1234,
    "preview": "...",  # 前 200 字
    "main_section": "...",  # 用户标记的"重点节"或全文
    "user_attempt": "...",  # 用户已尝试的解题步骤（很重要！）
    "vision_confirmed": True | False,  # source=vision_assisted 时必填；False → 禁止进入引导
}
```

> **`user_attempt` 字段是引导粒度的核心信号**：
> - 用户给了 attempt → 引导从 "卡在哪一步" 切入（中粒度）
> - 用户没给 attempt → 引导从 "复述题目 / 已知盘点" 开始（细粒度）
