# 苏格拉底式提问模板库

> 用于 Step 3 「引导问题」段的生成参考。**按场景 + 卡点类型查表**，避免每次现编。

---

## 通用 6 大反问类型

| 类型 | 句式 | 触发条件 |
|---|---|---|
| **复述确认** | "你能用自己的话复述一下题目要求吗？" | 用户读题不准 / 跳读关键词 |
| **已知盘点** | "你已经知道哪些条件？哪些是题目给的，哪些是你推出来的？" | 用户混淆"题给"和"自推" |
| **目标明确** | "你最终要得到什么？是数值、表达式、证明还是结论？" | 用户做着做着偏题 |
| **类比迁移** | "这跟你之前学过的 XXX 有什么相似 / 不同？" | 用户卡在"没见过这种" |
| **反向破题** | "如果结论已经成立，你会需要什么前提？" | 用户正向推不出 |
| **边界挑战** | "如果把条件 X 去掉 / 换成 Y，结论还成立吗？" | 用户机械记定理不理解 |

## 学科专题模板

### 数学（高数 / 线代 / 概率论）

- 「这个表达式的结构提示了什么变量替换？」
- 「分母的零点和被积区间什么关系？」
- 「定理的三个条件，你的题目都满足吗？哪一个最值得怀疑？」
- 「这个矩阵的特征值能告诉你什么直观信息？」
- 「这个分布的期望和方差，和题目要的"均值"是同一个东西吗？」

### 物理（力学 / 电磁 / 量子）

- 「画受力图了吗？哪个力你不太确定方向？」
- 「能量守恒和动量守恒，这道题哪个先用？为什么？」
- 「楞次定律的"反对变化"指向哪个量？」
- 「波函数 ψ 的归一化条件你写出来了吗？」

### 计算机（算法 / 数据结构 / OS / 网络）

- 「这个算法的时间复杂度，最坏 / 平均 / 最好情况各是多少？」
- 「红黑树的 5 条性质，违反了哪一条？」
- 「这个进程是 IO 密集还是 CPU 密集？调度策略应该怎么选？」
- 「TCP 三次握手少了一次会出什么问题？」

### 经济 / 管理

- 「这是边际分析问题还是均衡分析问题？」
- 「机会成本算了吗？沉没成本你是不是不该考虑？」
- 「需求弹性 > 1 还是 < 1，对企业定价意味着什么？」

### 论文写作（通用文科 / 理工科）

- 「这一段的核心 claim 是什么？一句话概括出来。」
- 「这条引用支撑了你的论证还是只是装饰？」
- 「读者读完这段，会问的下一个问题是什么？你回答了吗？」
- 「你的实验图能否在不读正文的情况下被理解？」

## 卡点类型 × 提问深度

### 卡点 A：读不懂题目

> 引导粒度：**最细**（拆题干）

- "题目里的 XXX 一词，是数学定义还是日常含义？"
- "求的是充分条件、必要条件还是充要条件？"
- "题目说『至少』『至多』『恰好』哪一个？"

### 卡点 B：知道大方向但卡某一步

> 引导粒度：**中**（指向那一步）

- "你前面推到 [用户最后一步]，下一步通常需要……（点名工具，不展开）"
- "这个变形的目的是什么？目标形式应该长什么样？"

### 卡点 C：完全没思路

> 引导粒度：**粗**（先给入口）

- "这道题属于哪一类问题？积分？证明？反证？构造？"
- "课本里有没有一个类似的例题？想想它是怎么开头的。"

### 卡点 D：算出来了但不放心

> 引导粒度：**校验式**

- "你能用一个特殊值代回去验证吗？"
- "结果的量纲对吗？数量级符合直觉吗？"
- "如果题干变一点，你的答案会跟着变成什么样？"

## 论文场景专题

### S1 选题

- "你导师组的近 3 篇代表作集中在哪？你想延续还是开辟？"
- "如果跑通你的设想，能得到什么样的一张图 / 一张表？倒推选题是什么。"
- "这个题目，3 年后再做还有意义吗？"

### S2 文献综述

- "你想用文献综述说服读者相信什么？"
- "你列的这 30 篇里，去掉哪 10 篇综述读起来反而更聚焦？"
- "你的『方法演进时间线』和『问题分类』哪条主线更适合你的论文结构？"

### S3 开题报告

- "问题陈述这一段，读者读完后能复述出『你要解决什么 + 为什么难』吗？"
- "你的研究目标和研究问题是不是同一件事？"

### S4 论证写作

- "这一段的 topic sentence 是哪一句？删掉它整段还能成立吗？"
- "支持论据和反方论据，你给了几行各自？"

### S5 修改 / 答辩

- "审稿人的这条意见，他真正想问的是『方法不对』还是『没说清楚』？"
- "答辩 PPT 第 3 页和第 4 页的逻辑跳跃，你打算怎么补？"

## 反模式（NEVER）

- ❌ 不要问 Yes/No 问题（"你会吗" / "懂了吗"）
- ❌ 不要问需要陈述完整答案的问题（"那答案是什么"）
- ❌ 同一段不要超过 3 个反问（认知超载）
- ❌ 反问之间不要相互独立（应该层层递进）
- ❌ 不要把"提示"伪装成"问题"（"你是不是该用换元法 u = x − 1/x？"——这是答案不是问题）

---

## 选模板的决策树

```
Step 1：用户的卡点是 A/B/C/D 哪一类？
  - A 读不懂题 → 粒度最细，从「复述确认 + 已知盘点」开始
  - B 卡某一步 → 中等粒度，「目标明确 + 类比迁移」
  - C 没思路 → 粗粒度，「类比迁移 + 反向破题」
  - D 不放心 → 校验式，「边界挑战」
Step 2：用户的学科领域？查学科专题模板
Step 3：是否论文场景？查 S1-S5 专题
Step 4：组合 1-3 个反问，确保层层递进
```
