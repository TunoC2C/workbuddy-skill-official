# 关键提示策略（Hints, Not Answers）

> 用于 Step 4 「关键提示」段的生成参考。**核心约束：给线索不给结论**。

---

## 6 种合规提示形态

### 1. 命名工具，不展开推导

```
✅ "这里需要用到拉格朗日中值定理"
❌ "由拉格朗日中值定理可得 f(b) − f(a) = f'(ξ)(b − a)"（展开了）
```

### 2. 给类比，不给映射

```
✅ "这个状态转移和爬楼梯问题很像，每步只能跨 1 或 2 级"
❌ "所以 dp[i] = dp[i−1] + dp[i−2]"（直接映射 = 给答案）
```

### 3. 给范围限定，不给具体值

```
✅ "这个分式的零点在 [0, π] 内"
❌ "在 x = π/2 处"
```

### 4. 给方向，不给步骤

```
✅ "尝试上下同除 x²，看看能不能让 (x − 1/x) 显形"
❌ "u = x − 1/x，du = (1 + 1/x²)dx，原积分变为 ∫du / (u² + 2)"
```

> ⚠️ **常见漂移**：把"提示"写成"换元 u = X 时，du = Y——和分子什么关系？"看似在反问，实则 du 已算好——属于本节 ❌ 范式。**正确做法是让用户自己算 du**（参考 SKILL.md NEVER 2 ✅ Good 示例）。

### 5. 给对照点，让用户自己迁移

```
✅ "这跟你昨天做的那道分部积分题结构上相似，区别在于多了一个三角因子"
❌ "所以套分部积分公式 ∫udv = uv − ∫vdu"
```

### 6. 给"为什么这么做"，不给"具体怎么做"

```
✅ "为什么要这一步换元？因为我们想把分母里的 x⁴+1 和分子的 x²+1 联系起来"
❌ "换元 u = x − 1/x 后……"
```

---

## 提示密度控制

| 用户档位 | 单次提示数 | 提示精度 |
|---|---|---|
| beginner | 3 条（多比喻多类比） | 给方向 + 给类比 + 给一个对照点 |
| intermediate（默认） | 2-3 条 | 命名工具 + 给方向 |
| advanced | 1-2 条（直接点关键转折） | 命名工具 + 给"为什么" |

## 论文场景的提示策略

### 选题阶段

✅ 给"判断框架"：
- "选题三角：问题真 × 方法新 × 资源够"
- "倒推法：先想能产出什么实验图"

❌ 不给"具体题目"：
- "你应该做'基于 CLIP 的小样本图像分类'"

### 综述阶段

✅ 给"组织维度"：
- "可以按方法演进时间线 / 按问题难度分类 / 按数据集评估"

❌ 不给"段落骨架"：
- "第一段写背景，第二段写方法 1……"（这是代写）

### 论证阶段

✅ 给"诊断切入点"：
- "topic sentence 在哪 / 反方论据在哪 / 跳跃点在哪"

❌ 不给"修改后的句子"：
- "可以改成『……』"

## 提示自检 Checklist

每写完一条提示，过一遍下面 4 个问题：

- [ ] 用户读完这条，**是否仍需要自己想一步**才能用？
- [ ] 这条提示**是否给了用户一个新的思考角度**而不是"开胃菜式答案"？
- [ ] 如果用户照这条提示做下去，**最后一步必须是用户自己合上**？
- [ ] 这条提示**是否避开了完整公式 / 完整证明 / 完整段落**？

> 4 个全 ✅ 才放行。任何一条 ❌，重写。

---

## 反模式（NEVER）

- ❌ 在提示里给完整公式 / 证明 / 段落
- ❌ 在提示里给最终数值 / 结论
- ❌ 把答案藏在第 3 条提示里换皮肤（NEVER 2）
- ❌ 提示完全脱离用户输入的具体题目（脑补）
- ❌ 提示之间互相重复（用 3 句话说同一件事）

---

## 极端情况：用户已重复要求"给答案"

参考 SKILL.md NEVER 3：

> 第 N 次后（默认 N=5）：**简化引导但不取消**。
> 把 3 条提示压成 1 条最关键的，仍要求用户做最后一步。

```
模板：
"我明白你急。一次性给到 80%——{关键转折，例如：'换元 u = x − 1/x 后，du 正好是分子'}。
你只需要做最后这一步：把它代入并积分，写下来发我。"
```
