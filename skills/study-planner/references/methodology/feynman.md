# 费曼学习法在学习中的应用

## 核心原理

诺贝尔物理学奖得主理查德·费曼：如果不能用简单的语言解释一件事，就说明你没有真正理解它。

费曼学习法四步：
1. **选择一个概念**
2. **假装教给一个小白**（写出解释）
3. **卡壳了？回头翻资料**（填补知识缺口）
4. **简化语言，重新组织**（用类比、比喻）

## 在学习计划中的应用

### 为什么每周要有"输出任务"

研究表明，**被动学习**（听/看）的知识留存率只有 10-30%，而**主动输出**（讲述/写作/应用）的留存率可达 75-90%。

| 学习方式 | 知识留存率 |
|---------|----------|
| 听讲 | 5% |
| 阅读 | 10% |
| 视听 | 20% |
| 演示 | 30% |
| 讨论 | 50% |
| 实践 | 75% |
| 教授他人 | 90% |

### 每周输出任务设计

建议每个计划中安排以下"输出型"任务：

| 频率 | 任务类型 | 说明 |
|-----|---------|-----|
| 每日 | 复盘笔记（3 句话） | 今天学了什么？哪里卡住了？明天重点？ |
| 每周 | 知识总结（1 页纸） | 用自己的话写下来，画思维导图 |
| 每周 | 讲解练习 | 对着镜子/手机讲一遍某个知识点 |
| 阶段结束 | 模拟讲解 | 假装你是老师，给"学生"讲一节课 |

### 复盘模板（每日 3 问）

```
📝 每日复盘

1. 今天学了什么？
   （用自己的话写，不要抄书）

2. 哪里卡住了？
   （标注模糊点，明天重点复习）

3. 明天要重点做什么？
   （1-2 条即可，不要贪多）
```

## 生成计划时的应用规则

1. 每天任务列表最后一条固定为「复盘笔记」
2. 每周至少安排 1 次「输出型任务」（讲解/写作/画图）
3. 每个阶段结束时安排「模拟讲解」或「全真模考」
4. 在计划中标注「📝 复盘」「🗣️ 讲解」「📊 模考」标签

## AI 注入指南

### 触发条件（满足任一即应用）

- 计划目标含「学懂」「真正掌握」「编程」「项目」「skill」
- 计划类型属于：编程/技能习得/学科理解（数学/物理/算法）
- 用户主动说"学了就忘""不会用""理解不深"

### 强度档位

| 档位 | 输出占比 | 适用 |
|-----|---------|------|
| 轻量 | 10% | 应试为主（每日 3 句复盘即可） |
| **标准** | 30% | 学科理解类 |
| 重度 | 60% | 编程/项目实战（example-react-60d 即此档） |

### 硬约束（生成时必须遵守）

1. 每天 daily_tasks 末尾固定 1 条 `category: "review"` 复盘任务
2. 每周至少 1 条 `category: "output"` 输出任务
3. 每个 stage 结束日必须有 `category: "demo"` 模拟讲解或模考

### 与其他方法论的协同

- **+ ebbinghaus**：每个复现节点搭配输出任务，记忆 + 理解双重加固
- **+ pareto**：高频考点优先输出（二八 × 费曼 = 高 ROI 输出）
- **+ pomodoro**：复盘任务 1 个番茄（25min），输出任务 2-3 个番茄
