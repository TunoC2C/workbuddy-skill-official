# 六大类票据识别指南（详细版）

> 配套：`SKILL.md` 六大类识别章节的详细参考。
> 适用：财务、志愿者、AI 助手在面对一张具体票据时，判断"这是什么票 / 该分到哪类 / 是否合规"。

---

## 使用方法

遇到一张具体票据时，**按以下顺序 5 步走**：

```
1. 看票头 —— 判断大类
2. 核必查字段 —— 判断是否合规
3. 查常见坑 —— 预判审计风险
4. 填台账 —— 按规则填哪些字段
5. 有异常 → 进《需人工复核清单》，不要硬塞进台账
```

---

## 第 1 类 · 增值税发票（纸质/PDF 均可）

### 1.1 子类

| 子类 | 票头文字 | 外观特征 |
|---|---|---|
| 增值税**专用**发票 | "XX省增值税专用发票" | **红色**套印边框、发票联+抵扣联+记账联三联 |
| 增值税**普通**发票 | "XX省增值税普通发票" | **蓝色**套印边框、发票联+记账联两联 |
| 卷式普通发票 | "XX省增值税普通发票（卷式）" | 热敏纸小票，加油站/超市常见 |

### 1.2 必查字段

- [ ] 发票代码（12 位）
- [ ] 发票号码（8 位）
- [ ] 开票日期（YYYY-MM-DD）
- [ ] **购买方名称 = 机构全称**（一字不差，核对营业执照）
- [ ] 购买方税号（18 位统一社会信用代码）
- [ ] 销售方名称 + 税号
- [ ] 货物或应税劳务名称
- [ ] 金额、税率、税额
- [ ] **价税合计**（大写 + 小写，两者必须一致）
- [ ] 开票人、收款人（专票还要有复核人）
- [ ] 发票专用章（盖在销售方位置）

### 1.3 常见坑

| 问题 | 后果 | 处理 |
|---|---|---|
| 抬头写个人名 | 不能报销、不能入账 | 退回让对方重开 |
| 税号错误或缺失 | 发票作废 | 退回重开 |
| 大写小写金额不一致 | 发票作废 | 退回重开 |
| 专票无销方税号/地址电话/银行账号 | 不能抵扣 | 退回重开（非抵扣场景可降级按普票处理）|
| 跨月后作废 | 只能红字冲销重开 | 联系对方开红字发票 |
| 章盖错位置/盖成财务章 | 不合规 | 退回重盖发票专用章 |

### 1.4 台账填法

- 类别列：`增值税-专票` 或 `增值税-普票`
- 日期列：开票日期
- 销方：销售方名称
- 金额：**价税合计（小写）**，不是"金额"也不是"税额"
- 用途：根据"货物或应税劳务名称" + 业务背景填写

## 第 2 类 · 电子发票（PDF）

### 2.1 识别特征

- PDF 格式，A4 版式
- 票头含"电子发票（普通发票）"或"电子普通发票"
- **无纸质红章**，但有电子签章（发票下方一个小图标）
- 有"发票校验码"（20 位）或"数电票号码"（20 位 "数电" 标识）

### 2.2 电子发票 vs 纸质发票的关键区别

| 维度 | 纸质增值税普票 | 电子发票 |
|---|---|---|
| 防伪 | 红色套印 + 发票专用章 | 电子签章 + 校验码 |
| 交付 | 邮寄 | 邮件 / 微信 / 开票方系统推送 |
| 报销 | 原件报销 | PDF 报销，**打印件仅作辅助** |
| 重复报销 | 物理唯一 | **必须校验发票号码是否已入账** |

### 2.3 必查字段

同第 1 类，额外加：

- [ ] **PDF 原件是否保存到云端**（只保留打印件不合规）
- [ ] **在台账中查重**（发票号码 + 金额 + 日期三项比对，防重复报销）

### 2.4 数电票（全电发票）的特殊字段

2024 年后普遍使用的"数电票"：
- 票号为 20 位（`24` 开头表示 2024 年开票）
- 无"发票代码"字段
- 购方信息可能只有名称+税号，无地址电话
- 在税务系统中**已自动入账**，机构只需校验 + 归档

## 第 3 类 · 公益事业捐赠票据

### 3.1 识别特征

- 票头"XX省公益事业捐赠票据"（**不是"发票"**）
- 财政部监制章（深红色，印在票据右上角）
- 有"捐赠票据字轨"（如 `粤捐字 001 号`）
- 有"票据号码"（8 位）

### 3.2 依据

- 《公益事业捐赠票据使用管理办法》财综〔2024〕1 号
- 详细条文见 `references/05-公益事业捐赠票据使用管理办法（财综2024-1号）.md`

### 3.3 必查字段

- [ ] 捐赠人名称（个人 → 姓名；企业 → 全称+统一社会信用代码）
- [ ] 捐赠日期
- [ ] 捐赠金额（大写 + 小写一致）
- [ ] **捐赠用途**（指定项目时必须明确项目名称；未指定写"非限定"）
- [ ] 受赠单位公章
- [ ] 经办人签字
- [ ] 票据字轨 + 号码
- [ ] 是否已交付给捐赠人（电子票据系统可查送达状态）

### 3.4 审计红线

🚨 **以下是审计必查、违反即不合规的事项**：

1. **不得跳号**：票据号码必须连续，跳号必须有《作废票据报备表》存档
2. **不得串用**：捐赠票据只能开捐赠收入，不能开服务费/咨询费
3. **不得代开**：只能为本机构捐赠收入开票，不得为其他机构代开
4. **作废票据保存全联**：加盖"作废"戳记，所有联次一起存档
5. **不得转让出借**：不能把空白票据借给别的机构
6. **月度核销**：定期向财政部门申请核销，否则下年不予发票

### 3.5 台账填法

- 类别列：`捐赠票据`
- 销方列：**改填"捐赠人"**
- 金额：捐赠金额
- 用途：指定项目名称 / "非限定"
- 备注：票据字轨+号码 + 是否开具电子副本

## 第 4 类 · 差旅报销

### 4.1 子类

| 子类 | 识别特征 |
|---|---|
| 火车票 | 蓝色磁介质票（实名制）或 12306 打印的"电子客票行程信息" |
| 飞机行程单 | A4 竖版，"航空运输电子客票行程单"字样，含航班号、舱位 |
| 高铁/动车 | 同火车票（但金额和座位类型不同） |
| 出租车票 | 热敏纸小票，有起始/结束时间、里程、金额 |
| 网约车（滴滴/高德） | 电子发票（走第 2 类）或行程单（需附合规发票） |
| 住宿发票 | 属第 1/2 类增值税发票，但用途为差旅 → 按差旅归 |

### 4.2 必查字段

- [ ] 乘车/飞行/入住日期 → **必须落在出差审批单时间范围内**
- [ ] 起讫地点/航线 → **必须和出差目的一致**
- [ ] 票价金额
- [ ] 旅客/入住人姓名 → **必须是本机构员工**
- [ ] 舱位/铺位等级 → **不得超过机构差旅标准**

### 4.3 常见坑

| 问题 | 风险 | 处理 |
|---|---|---|
| 周末/节假日的票 | 审计会质疑 | 附加说明（加班/项目紧急） |
| 改签/退票费 | 影响报销金额 | 附改签凭证，说明原因 |
| 一张订单多人 | 需要拆分 | 附乘客名单 |
| 火车票丢失 | 不能报销 | 12306 重新打印行程信息 + 附说明 |
| 滴滴打车只有行程单无发票 | 不合规 | 在滴滴 APP 补开电子发票 |
| 乘车人不是本机构员工 | 不得报销 | 退回经办人 |

### 4.4 台账填法

- 类别列：`差旅-火车` / `差旅-飞机` / `差旅-出租` / `差旅-住宿`
- 日期列：**出行日期**（不是开票日期）
- 销方：承运人（如"中国国际航空")
- 金额：含税总价
- 用途：**必须填"XX 项目出差：起讫地-目的地"**，不能只写"出差"

## 第 5 类 · 餐饮

### 5.1 识别特征

- 含"餐饮服务""餐费""食品"等字样的增值税发票
- 或餐厅打印的小票（**非正规发票 → 不能单独报销，需补开发票**）

### 5.2 必查字段 + 附件

餐饮票的特点是**必须附加说明文件**，单票不能报销：

- [ ] 用餐日期
- [ ] 金额（税价合计）
- [ ] 人均消费（金额 ÷ 参会人数）
- [ ] **附：会议议程 / 参会人名单 / 活动记录 / 业务接待说明**

### 5.3 合规边界

| 场景 | 合规判断 |
|---|---|
| 项目会议 / 志愿者集体用餐 | ✅ 可报销，附议程+名单 |
| 与资助方/合作方洽谈 | ✅ 可报销，附业务说明；超限需特批 |
| 员工团建 | ⚠️ 看机构福利制度，可能要走管理费用 |
| 个人请客 / 员工家庭聚餐 | ❌ 不得报销 |
| 超过人均 200 元的 | ⚠️ 大多数公益机构的餐饮标准以内，超标需特批 |

### 5.4 台账填法

- 类别列：`餐饮`
- 用途：**具体到会议名称 + 人数**，如"2026-04 项目启动会议，12 人"
- 备注：是否附议程 / 名单

## 第 6 类 · 其他

### 6.1 常见子类

| 子类 | 处理方式 |
|---|---|
| 白条（手写收条） | 不得作为正式凭证，除非是受益人/志愿者补贴签收表 |
| 境外消费票据 | 附中文翻译 + 当日汇率换算 + 中央银行汇率证明 |
| 签收表（受益人/志愿者补贴） | 按场景四 4.4 模板填写，附身份核验 |
| 合同/协议 | **不属于票据**，单独建"合同档案" |
| 银行手续费 | 看银行流水自动入账，无需单独票据 |
| 工资、社保、公积金 | 走"工资社保台账"，不进票据台账 |
| 押金 / 保证金 | 先进"其他应收款"，收回时冲销，不进费用台账 |

### 6.2 处理原则

- **默认全部进《需人工复核清单》**
- 不要自动计入台账合计
- 需要财务负责人审核后决定归属

## 附录 A：各省公益捐赠票据样式差异

不同省份财政厅监制的捐赠票据**版式略有差异**，但关键字段一致。常见差异：

- 广东：票据字轨 `粤捐字`，票据右上角财政部监制章为深红色圆章
- 北京：票据字轨 `京捐字`，版式更接近收据
- 上海：电子捐赠票据占比高，可从"公益事业捐赠票据查询平台"验真
- 浙江：票据右下角有二维码可扫码验真

**通用验真方法**：到受赠地省级财政厅官网的"公益事业捐赠票据查询"页面，输入票据字轨+号码+金额即可验真。

## 附录 B：AI 辅助识别时的置信度标准

AI 在路径 B 中识别票据时，按以下标准决定进"已识别"还是"需复核"：

| 字段 | 高置信度（进已识别） | 低置信度（进需复核） |
|---|---|---|
| 票据类型 | 票头文字清晰可见 | 票头模糊 / 只看到一部分 |
| 销方/捐赠人名称 | 完整识别，字符无 OCR 错字 | 有乱码或疑似错字 |
| 日期 | 格式标准（YYYY-MM-DD 可解析） | 日期缺失 / 格式异常 |
| 金额 | 大写+小写都识别且一致 | 任一缺失 或 两者不一致 |
| 发票号码 | 12+8 位或 20 位完整 | 位数不对 / 有 OCR 错字 |

**只要任一核心字段（票据类型/日期/金额/发票号）走低置信度，整张票进《需复核清单》。**
