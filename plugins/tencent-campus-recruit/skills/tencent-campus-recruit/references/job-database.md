# 岗位方向面试官考察视角

> 具体岗位名称、招聘类型、工作地、JD 全文优先走 `scripts/fetch_recruit_jds.py` 从 `join.qq.com` 官网接口获取；仅受控内测且 JD MCP 可用时，可选用 `campus-recruit-jd-qa` 辅助检索。
> 本文件仅提供各大方向的面试官考察视角，用于辅导同学准备。

> ⚠️ **输出约束**：引用本文件内容时，禁止使用"一定会考察""肯定看重"等绝对化表述。统一使用"通常比较关注""往往会看重""建议重点准备"等柔性措辞。

> ⚠️ **通过率禁令**：禁止编造或推测任何岗位的通过率、录取率、淘汰率、竞争比等数据。我们没有接入此类后端数据。当同学询问时，引导关注「腾讯招聘」公众号发布的热招岗位信息。

> ⚠️ **岗位推荐前置**：任何涉及"岗位推荐 / 在招什么 / 某岗位 JD / 某方向有什么 / 某城市能投什么"的场景，优先运行 `scripts/fetch_recruit_jds.py search/detail/match` 从 `join.qq.com` 官网接口获取真实岗位；仅受控内测且 JD MCP 可用时，可选用 `campus-recruit-jd-qa` 辅助检索。必须基于真实返回岗位回答，不凭训练数据推荐。

> ⚠️ **院校中性约束**：岗位匹配、JD 解读或结合简历背景给建议时，如涉及学校层级、院校标签或学历背景，只能作为客观信息参考；不得以学校层级给人选贴标签、排序或判断优劣，匹配理由应聚焦专业、项目、技能、经历、城市和实习类型等可说明因素。

## 使用说明

1. 当用户询问"在招什么岗位 / 有什么 JD / 推荐岗位" → 优先走 `scripts/fetch_recruit_jds.py` 官网接口；仅受控内测且 JD MCP 可用时，可选用 `campus-recruit-jd-qa` 辅助。只列 3-5 个与同学简历/背景最符合的真实校招岗位。推荐结果统一用紧凑表格输出，列为 `# / 岗位名称 / 工作地 / 匹配理由 / 投递链接`；岗位名称加粗；岗位名称、工作地、投递链接必须来自官网脚本或可选 JD MCP 的真实返回；匹配理由可写 2-3 个短点并用 `<br>` 在同一单元格内换行；投递链接统一显示为 `[点击投递](url)`；不输出评分、分数或匹配度百分比
2. 当用户给具体岗位名 → 先用 `fetch_recruit_jds.py search` 找 `post_id`，再用 `fetch_recruit_jds.py detail <post_id>` 拿官网完整 JD；仅受控内测且 JD MCP 可用时，可选调 `get_jd_detail`；再结合本文件对应方向的面试官视角做辅导
3. 当用户想了解某个方向的面试准备 → 参考本文件对应方向的面试官视角
4. 本文件不替代官方 JD，仅作为辅导同学时的方向性参考材料

## 技术方向

### 面试官怎么看
> "看基本功是否扎实。也会问一些超纲的问题来判断思维速度和清晰度。看重学习能力。倾向于对自己有清晰定位的同学。"

### 通常考察维度
- **专业基础**：计算机基础知识的扎实程度
- **代码能力**：实际动手写代码、解决问题的能力
- **工程素养**：代码质量意识、系统设计思维
- **学习能力**：快速掌握新技术的能力
- **态度热情**：对技术的热爱和钻研精神

### 面试形式参考
基础知识问答 + 代码/工程题 + 项目深挖 + 经历问答

### 简历建议
- 项目经历是核心，用STAR描述
- 技术栈列清楚，但要有使用场景佐证
- 竞赛/论文/开源贡献是加分项

## 产品方向

### 面试官怎么看
> "关注他是否喜欢做产品，这是走得更远的核心动力。能找到明确目标，有承压阈值。思考和表达都有逻辑性。团队协作有感知度。"

### 通常考察维度
- **产品热情**（面试官普遍很看重）：是否真正喜欢做产品
- **逻辑思维**：分析问题是否有条理
- **用户洞察**：能否站在用户角度思考
- **沟通表达**：能否清晰表达想法
- **团队协作**：对分工协作有感知度

### 面试形式参考
经历问答 + 案例分析（用真实产品案例讨论）

### 简历建议
- 突出产品思维和用户洞察
- 数据驱动的案例比职责描述更有说服力
- 即使没有实习经验，也可以通过案例分析展现潜力

---

## 游戏方向

### 面试官怎么看
> "特别看重的是他自己真的是深度游戏玩家，要玩过定义品类的支柱性作品。动手能力也很重要——做过哪些游戏？用过哪些引擎？策划越来越技术向了。"

### 通常考察维度
- **热爱度**（面试官普遍很看重）：是否是深度玩家
- **游戏理解**：对游戏设计、品类的深入理解
- **动手能力**：有没有自己做过游戏或demo
- **转化能力**：能否把想法变成可执行的方案
- **适配度**：和团队、方向的匹配程度

### 面试形式参考
经历问答 + 案例分析 + 专业笔试 + 项目demo展示

### 简历建议
- 游戏经历和深度体验往往是面试官很看重的
- 自己做过的demo、用过的引擎要突出写
- 展示对游戏品类的深入理解和热爱

## 市场/职能方向

### 面试官怎么看
> "产出细节要过硬，至少看出是认真在做的，不是好高骛远。开放学习心态。不要过早局限自己，更欣赏一直在探索的同学。"

### 通常考察维度
- **执行力**：做事的细节和质量
- **学习心态**：是否开放、好奇、主动
- **探索精神**：不局限于一个角色
- **沟通能力**：表达清晰，善于协调

### 面试形式参考
经历问答 + 专业笔试（部分岗位）+ 案例分析 + 情景模拟（部分岗位）

### 简历建议
- 突出实际产出的细节和质量
- 展示学习心态和探索精神
- 有数据支撑的案例优于泛泛描述
