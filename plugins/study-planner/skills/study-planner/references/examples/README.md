# Few-shot 优秀计划示例

本目录提供 3 个**高质量 plan.json 示例**，作为 AI 生成计划时的参照标杆。
每个示例都包含 `_example_meta` 字段说明它的设计要点，AI 在生成时应：

1. **结构对齐**：JSON 字段、嵌套、ID 命名规范完全保持一致
2. **风格借鉴**：任务 title 写法、duration 估算、methodology_tip 的位置
3. **场景匹配**：根据用户场景找最近似的示例做"增强参照"

> ⚠️ `_example_meta` 字段**仅用于 AI 学习**，生成真实计划时**必须删除**。

## 🎯 三个示例覆盖的"模式光谱"

```
                  短期冲刺            长期备考           技能习得
                  雅思 30d           考研 120d          React 60d
─────────────────────────────────────────────────────────────────
任务密度          高（每日3-4项）    极高（每日5-6项）   中（每日2-3项）
exam 占比         40%                30%                 5%
output 占比       10%                5%                  60%
review 占比       20%                25%                 10%
methodology 重点  ebbinghaus         pomodoro+pareto     feynman
模考节奏          每周 1 次          每月 1 次后期加密   无（替换为交付）
日预算            90/180             360/480             90/240
```

AI 在生成时应判断用户场景落在光谱哪个位置，靠近哪个示例即重点参照之。

## 📐 关键约束（生成时必须遵守）

| 约束 | 说明 |
|-----|------|
| **任务粒度** | 单任务 25-90min，超过则拆分 |
| **每日任务数** | 工作日 ≤ 5 项，周末 ≤ 7 项 |
| **category 严格枚举** | 只能用 schema 定义的 10 类 |
| **艾宾浩斯标注** | 词汇 / 概念类任务必须在 methodology_tip 注明 D+1/D+3/D+7 复现点 |
| **周日 review** | 所有计划必须每周日固定 1 个 review 任务，不安排新内容 |
| **薄弱项加权** | weak_points 中的领域，工作日**每天**至少 1 个相关任务 |
| **阶段切换日** | 阶段切换当天用 1 个特殊任务（emoji 🎯）做承上启下 |
| **冲刺末期降负** | 考前 3 天主动降低强度，避免过度训练 |

## 🔄 AI 使用流程

```
用户场景 → 匹配模板（templates/） → 找最近似 example
                                       ↓
                              抽取风格特征
                                       ↓
                  根据用户的 deadline / level / budget
                  生成结构相同、内容定制化的 plan.json
                                       ↓
                          删除 _example_meta 字段后保存
```

## ✏️ 增量贡献

如果用户的某个高质量计划具有特殊参考价值，可以：

1. 将其 `plan.json` 复制到本目录
2. 在文件顶层加 `_example_meta` 字段说明亮点
3. 更新本 README 索引
4. 重命名为 `example-{scenario}-{duration}.json`
