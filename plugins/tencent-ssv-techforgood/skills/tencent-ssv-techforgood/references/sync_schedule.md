# 数字工具箱惰性更新机制（Lazy Refresh）

> **更新日期**：2026-05-15
> **用途**：定义本地产品快照 `digital_toolbox.md` 的惰性更新规则——只在用户实际使用 Skill 时检查时间戳，超过阈值就在 Step 3 流程中透明完成更新，不创建后台定时任务、不打扰用户。

## 一、设计原则

- **惰性触发（Lazy）**：只在用户实际进入 Step 3（产品抓取）时检查快照新鲜度，从不主动后台运行
- **零打扰**：用户感知不到"在更新"——更新过程复用 Step 3 本来就要做的 `web_fetch`，无额外耗时
- **零依赖**：不依赖任何 IDE 自动化任务、CI 服务或外部调度，单凭 Skill 内部规则即可工作
- **失败保护**：更新失败时不写回本地，沿用旧快照继续服务，不影响当前对话

## 二、新鲜度判定规则

### 2.1 时间戳来源

`references/digital_toolbox.md` 首部的 `**快照日期**：YYYY-MM-DD` 字段。

### 2.2 阈值定义

| 距离今日 | 状态 | 行为 |
|---|---|---|
| ≤ 14 天 | 🟢 新鲜 | 直接使用本地快照兜底，不触发更新 |
| 15–30 天 | 🟡 偏旧 | 在线抓取成功后，**顺手回写**本地快照 |
| > 30 天 | 🔴 过期 | 在线抓取后**强制回写**；如在线失败，向用户提示"本地数据已 N 天未更新，建议核对官网最新版本" |

### 2.3 触发时机

**唯一触发点**：Step 3（产品数据抓取阶段），且仅在以下条件之一满足时：
- 在线抓取（`tools/productList?zone=zone-1` + `tools/digitalServices`）通过完整性判断
- 距离上次快照日期超过 14 天

不触发的场景：
- 用户在 Step 1/2/4/5/6（不涉及产品数据抓取）
- 在线抓取失败或不完整（此时只兜底使用，不污染本地）
- 距离上次快照日期 ≤ 14 天且本次为兜底兜底场景（避免无意义写入）

## 三、Step 3 中的执行流程（增量补丁）

在 `workflow_steps.md` 既有的 Step 3 流程基础上，**插入两个动作**：

```text
[Step 3 既有流程]
1. 输出"🦞 公益虾正在从腾讯技术公益数字工具箱为您匹配最合适的工具，请稍候..."
2. web_fetch(productList) + web_fetch(digitalServices)
3. 完整性判断
4. （惰性更新增量 A）→ 读取 references/digital_toolbox.md 首部"**快照日期**"
5. 计算 ageDays = 今日 - 快照日期
6. （惰性更新增量 B）→ 若 ageDays > 14 且在线抓取通过完整性判断
       → 用在线结果重写 references/digital_toolbox.md
       → 同步更新文件首部"**快照日期**"为今天
       → 在用户对话中**无需特别提示**（透明完成）
   若 ageDays > 30 且在线抓取失败
       → 在用户对话末尾用一句话提示："📌 本地产品快照已 N 天未更新，部分产品信息可能滞后，建议直接核对官网 https://techforgood.qq.com/tools/productList?zone=zone-1"
       → 不阻塞当前推荐流程
7. 继续 Step 4 智能匹配
```

## 四、回写规范

更新本地快照时必须遵守的规则：

1. **结构保持**：分区顺序、表格列结构、章节编号不变（详见 `digital_toolbox.md` 既有格式）
2. **路径稳定**：腾讯自有产品 `tools/toolsDetail/{id}`，伙伴产品 `tools/digitalService/{id}`，外链产品保留官方域名
3. **快照日期更新**：首部 `**快照日期**` 字段更新为今天的 `YYYY-MM-DD`
4. **保留误删保护清单**：第七节"误删保护清单"中的产品（如腾讯云智绘、音书-手语翻译服务基础版）若本次未在汇总页出现，**不删除**，沿用上次快照
5. **大变动告警**：若新增/下架产品 ≥3 个，在用户对话末尾用一句话告知："📌 本次更新发现工具箱有较大变动（新增 X / 下架 Y），完整列表已同步至本地快照"

## 五、误删保护清单

以下产品历史上曾因运营页面延迟更新被误删，惰性更新时如发现这些产品在本轮抓取的汇总页中"消失"，**必须保留**，不删除：

| 产品 | 详情页路径 | 复盘备注 |
|---|---|---|
| 腾讯云智绘 | `tools/toolsDetail/23` | 2026-05-15 复盘：曾因汇总页未列被误删 |
| 音书-手语翻译服务基础版 | `tools/digitalService/47` | 2026-05-15 复盘：曾因汇总页未列被误删 |

**保留逻辑**：
- 单次抓取中没出现 → 保留本地条目，不删除
- 连续 3 次以上抓取都没出现 → 在对话末尾建议人工二次确认（仍不自动删除）
- 直接 `web_fetch` 详情页 URL 验证产品已下线 → 才允许人工手动从快照中移除

## 六、写入失败保护

- `replace_in_file` / `write_to_file` 失败 → 沿用本地旧快照继续服务，不影响当前对话；下次用户使用时再次尝试
- 写入过程中网络中断 → 不会破坏现有快照（`write_to_file` 是原子覆盖）
- 完整性判断未通过 → 直接跳过写回步骤，本轮只用在线结果做推荐

## 七、为什么不用后台定时任务

- ❌ 创建 IDE 自动化任务会**在用户不使用时也定期执行 web_fetch**，无端消耗资源
- ❌ 用户长期不用该 Skill 时，更新没意义
- ❌ 桥接服务可能不稳定（已观察到 Agent Manager automation bridge unavailable）
- ✅ 惰性更新 = 用户用时才更新 = 资源最小化、影响最小化、可靠性最高
