# 自检护栏

> **更新日期**：2026-05-15
> **用途**：每个 Step 输出前做最小必要自检。

## 一、Step 级自检

### Step 1
1. **首屏响应**：是否**先输出问候文案**再调用 `web_search`/`web_fetch`？（不要让用户为搜索等待 8-12s 才看到第一句话）
2. 渠道交互方式正确？（IDE 用 `AskUserQuestion`；IM 用编号文本）
3. 路径 B 只采集 `org_type` / `org_field` / `org_size`？
4. 路径 A 预填坚持"官网优先，其他来源待确认"？
5. 路径 A 全程**最多 1 次 `web_search` + 最多 1 次 `web_fetch`**？没有连续抓多个第三方源（百科+知乎+...）？
6. 路径 A 的 `web_search` 超过 5s 未返回稳定结果时，是否已降级到路径 B？
7. 没有提前推荐产品或读取产品快照？
8. 本轮已收口等待用户回复？

### Step 2
1. 渠道交互方式正确？
2. 只采集 `pain_points` 和 `urgency`？
3. 先复述了 Step 1 的机构信息？

### Step 3（Local-First）
1. 是否**先读了** `references/digital_toolbox.md` 并提取了快照日期？（不要跳过这一步直接联网）
2. 是否正确计算了 `ageDays` 并按决策树判断"是否需要联网"？
   - ageDays ≤ 14 且用户没要求"最新" → **不联网**
   - ageDays > 14 或用户要"最新" → 联网
3. 联网时是否只用了两个固定 URL（`tools/productList?zone=zone-1` 与 `tools/digitalServices`）？没有用 `web_search`？
4. 联网时是否做了完整性判断？满足以下任一条件已用本地快照兜底/补齐？
   - 任一 URL 抓取失败
   - 汇总页有效产品少于 **30** 个，或缺少 `腾讯文档` / `企业微信` / `腾讯会议` / `腾讯WorkBuddy` / `腾讯混元大模型` 中任一关键产品
   - 伙伴产品页有效产品少于 **8** 个，或缺少 `益帮帮-项目执行管理系统` / `金数据公益企业版` / `灵析-联系人管理系统标准版` 中两项及以上
   - 页面只有导航/营销文案/零散片段，无法形成稳定产品清单
5. 联网兜底时是否保留了在线结果中的实时字段，只补本地缺失字段？
6. 仅使用本地快照时（无论是 L0/L1/L2/L3 哪一级），是否已向用户透明披露数据源（"基于本地快照（{快照日期}）"）？
7. ageDays > 14 且联网完整成功时，是否已用 `write_to_file` 回写了本地快照（透明完成，不打扰用户）？

### Step 4
1. 推荐产品全部来自 Step 3 获取的工具箱数据（本地快照或在线抓取，不是外部产品）？
2. **推荐数量 ≤ 6 款？**（超过会让 AskUserQuestion 选项卡塞不下，必然导致前后不一致）
3. **🔴 推荐文本（表格/摘要）中的所有产品 ≡ 选项卡中的所有选项？**（铁律：文本列了 N 款，选项卡也必须有 N 个对应选项；图中"文本 10 款 / 选项卡 5 款"的错位绝对禁止）
4. 选项卡末尾是否包含 `📋 查看其他可选工具` + `🔙 返回修改我的需求` 两个固定兜底项？
5. 推荐策略与 Step 2 的 urgency 匹配？（🚀 立刻可用→3-4 款 / 📅 这周内开始→5-6 款 / 🗺️ 先了解全貌→6 款）
6. 遵循"领域相关 → 免费腾讯产品优先 → 第三方伙伴产品"排序？
7. 分析摘要中**没有再次列产品**（避免与选项卡重复，造成用户信息轰炸）？

### Step 5
1. 每张详情卡包含：推荐理由 / 产品详情 / 申领入口？
2. 教程/案例/实施参考先经 `references/knowledge_index.md` 路由，从在线总表实时生成？
3. 申领地址只核对 `references/digital_toolbox.md`？
4. 产品没有稳定独立详情页时，如实说明使用统一入口，不编造隐藏直链？
5. 链接格式符合渠道要求？（IDE 超链接；IM 纯文本 URL）

### Step 6
1. 输出了清单速览？
2. 每个推荐产品都有申领入口？
3. 附上了申领须知？
4. 发起了后续操作交互？

## 二、全局自检

1. **渠道一致性**：IDE 全程交互工具 + Markdown 表格；IM 全程编号文本 + 纯文本 URL
2. **主线一致性**：只围绕公益机构数字化赋能
3. **收尾完整性**：推荐过工具就必须有清单速览
4. **敏感信息**：涉及身份证号/手机号/签字盖章件/AppSecret/AppCode 时附安全提醒

## 三、法规引用自检

1. 先查权威来源（`flk.npc.gov.cn` / `mca.gov.cn`），不依赖本地静态法规快照
2. 引用法规必附：`📌 以上法律法规信息仅供参考，不构成法律意见。请以 flk.npc.gov.cn 最新版本为准。`
3. 涉及比例/门槛/时间节点时给出具体数字和条文依据
4. 无法实时核验时，明确提示用户以国家法律法规数据库及主管部门最新发布为准
